СУБД SQLite представляє собою сховище, яке складається з одного файлу. Ця СУБД не має робочих
процесів, запущених та очікуваних з'єднань, як, наприклад, MySQL або PostgreSQL.

SQLite підтримує типи даних для кортежів значень, але кількість типів обмежена, а також існує
таке поняття, як афінованість типів. Афінованість використовується для сумісності SQLite з різними
SQL СУБД, використовуючи такі типи даних – Integer, Text, None, Real, Numeric.

У стандартної бібліотеки мови Python є модуль sqlite3, який призначений для роботи з даною СУБД.
Він дозволяє виконувати SQL-запити, які описуються як звичайні рядки, мають тип str. Також є
можливість створювати користувацькі агрегатні та звичайні функції, а також власні типи, реєструючи
їх через спеціальний інтерфейс бібліотеки sqlite3.

У світі SQL СУБД існує такий вид загрози або атаки, що називається SQL-ін'єкцією. Даний вид загрози
дозволяє вбудовувати шкідливий SQL-код у запити, якщо ми не використовуємо екранування символом при
вставці змінних SQL запити враховуючи типи їх значень. sqlite3 надає можливості для запобігання таких
атак.

SQLite
- Компактність.
- Читають усі, а пише один.
- відкритий вихідний код.
- Стандарт SQL 92.
- афіновані типи.
- Кросплатформність.

import sqlite3
connection = sqlite3.connect(“db.sqlite3”)
cursor = connection.cursor()
result = cursor.execute(“SELECT id, email FROM USERS”)
